Inicio del Repaso Práctico de JavaScript

Este repaso está diseñado para introducirte a conceptos clave de JavaScript, fundamentales para el desarrollo con React. Estos conceptos incluyen funciones modernas, manejo de asincronía y la estructura de código reutilizable, que son pilares en la creación de componentes en React.

Funciones

  • Definición Ampliada: Las funciones son bloques de código reutilizables que realizan una tarea específica.
  • Uso: Base del desarrollo modular. En React se usan para crear componentes funcionales.
  • Cuándo es necesario: Al agrupar instrucciones que se repiten o para dividir el código.
  • Ventaja: Reutilización y legibilidad del código, además de facilitar la depuración.

Funciones

Una función en JavaScript es un bloque de código reutilizable que realiza una tarea específica.

¿Para qué se usan? Para encapsular lógica y dividir un programa en partes pequeñas.

function saludar(nombre) {
  return "Hola " + nombre;
}

Función flecha (Arrow Function)

  • Definición Ampliada: Sintaxis concisa de funciones introducida en ES6.
  • Uso: Popular en React para componentes y eventos.
  • Cuándo es necesario: Cuando se necesita preservar el contexto this.
  • Ventaja: Sintaxis corta y manejo automático del this.

Función Flecha

Las funciones flecha son una forma más corta de escribir funciones.

¿Dónde se usa en React? En eventos, efectos, map, filtros, y componentes simples.

const saludar = nombre => `Hola ${nombre}`;

Callback (Función de Retorno de Llamada)

  • Definición Ampliada: Función pasada como argumento y ejecutada al finalizar otra función.
  • Uso: En eventos, setTimeout, APIs, etc.
  • Cuándo es necesario: Para tareas asíncronas o con espera.
  • Ventaja: Permite asincronía sin bloquear el flujo del programa.

Callback

Un callback se pasa a otra función y se ejecuta cuando termina una operación.

Ejemplo típico:

function mostrarResultado(callback) {
  const resultado = 5 + 3;
  callback(resultado);
}

mostrarResultado(res => {
  console.log("El resultado es", res);
});

Promesas

  • Definición Ampliada: Objeto que representa el resultado de una operación asíncrona.
  • Uso: Para manejar tareas como fetch, lectura de archivos, etc.
  • Cuándo es necesario: Para tareas que toman tiempo y requieren respuesta.
  • Ventaja: Mejor estructura, evita "callback hell" y facilita manejo de errores.

Promesas

Una promesa representa un valor futuro, con métodos para manejar éxito y error.

¿Para qué sirve? Para manejar tareas asincrónicas como llamadas a APIs.

fetch("https://api.com/datos")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

Async/Await

  • Definición Ampliada: Sintaxis moderna para trabajar con Promesas.
  • Uso: Hace el código asincrónico más legible.
  • Cuándo es necesario: Para evitar .then() y usar try...catch más intuitivo.
  • Ventaja: Mejora la depuración y la legibilidad del código.

Async / Await

async y await simplifican el uso de promesas.

Ventajas: Código más claro y estructurado.

async function obtenerDatos() {
  try {
    const res = await fetch("https://api.com/datos");
    const data = await res.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
